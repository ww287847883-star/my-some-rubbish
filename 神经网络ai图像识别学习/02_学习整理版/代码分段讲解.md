# Keras CT 三维图像识别代码分段讲解

对应文件：`ct_3d_classification_learning.py`

## 1. 这个代码要解决什么问题

它用 3D CNN 判断一个胸部 CT 是否存在病毒性肺炎相关异常。输入不是普通图片，而是一个 CT 体数据，也就是很多张切片堆起来的三维数组。

任务类型：二分类。

- `0`：正常 CT
- `1`：异常 CT

## 2. 导入依赖

代码开头导入了：

- `numpy`：处理数组
- `tensorflow`：构建数据管道
- `keras`：搭建和训练神经网络
- `nibabel`：读取医学影像 `.nii` 文件
- `scipy.ndimage`：旋转、缩放三维图像
- `matplotlib`：画图和显示 CT 切片

## 3. 下载 CT 数据

源码会下载两个压缩包：

- `CT-0.zip`：正常肺部 CT
- `CT-23.zip`：有异常表现的 CT

下载后解压到 `MosMedData` 文件夹。

注意：数据体积比较大，第一次运行会花时间，并且需要网络。

## 4. 读取 `.nii` 医学影像

函数：

```python
def read_nifti_file(filepath):
```

作用：用 `nibabel` 读取 CT 文件，并转成三维数组。

医学 CT 常见格式不是 `.jpg` 或 `.png`，而是 `.nii`、`.nii.gz`、`.dcm` 等。这个示例用的是 `.nii`。

## 5. CT HU 值归一化

函数：

```python
def normalize(volume):
```

CT 图像的像素值通常是 HU 值，不是普通图片的 0 到 255。代码把 HU 限制在：

- 最小：`-1000`
- 最大：`400`

然后缩放到 `0` 到 `1`。这是医学图像预处理中很重要的一步。

## 6. 调整 CT 三维尺寸

函数：

```python
def resize_volume(img):
```

模型需要固定输入尺寸，所以代码把每个 CT 统一调整成：

```text
128 x 128 x 64
```

含义：

- 宽：128
- 高：128
- 深度：64 张切片

## 7. 组成训练集和验证集

代码把正常和异常 CT 合并，然后分成：

- 训练集：140 个样本
- 验证集：60 个样本

标签规则：

```python
abnormal_labels = 1
normal_labels = 0
```

## 8. 数据增强

函数：

```python
def rotate(volume):
```

训练时会随机旋转 CT 体数据，让模型见到更多变化，减少过拟合。

## 9. 构建 3D CNN

函数：

```python
def get_model(width=128, height=128, depth=64):
```

普通 CNN 用 `Conv2D` 处理图片；CT 体数据用 `Conv3D`，因为它同时看宽、高、切片深度三个方向。

模型主要结构：

- 多层 `Conv3D`
- 多层 `MaxPool3D`
- `GlobalAveragePooling3D`
- 全连接层
- 最后输出一个概率

最后一层：

```python
layers.Dense(units=1, activation="sigmoid")
```

表示二分类概率。

## 10. 训练模型

核心代码：

```python
model.compile(...)
model.fit(...)
```

使用：

- 损失函数：`binary_crossentropy`
- 优化器：`Adam`
- 指标：`accuracy`

## 11. 查看训练效果

代码会画出：

- 训练准确率
- 验证准确率
- 训练损失
- 验证损失

如果训练准确率很高但验证准确率很低，就是过拟合。

## 12. 单个 CT 预测

最后一段代码会拿一个验证集 CT 做预测，输出它属于正常或异常的概率。

## 13. 建议你重点改哪些地方

初学阶段建议只改这些：

- `batch_size`
- `epochs`
- `learning_rate`
- `desired_width`
- `desired_height`
- `desired_depth`

不要一开始就大改模型结构。先跑通，再一点一点实验。
